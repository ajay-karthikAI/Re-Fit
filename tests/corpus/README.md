# Test corpus

Real-world fixture files for the parse → tailor → score → render pipeline.

- `resumes/` — real resume files as users would actually upload them: PDF and
  DOCX, including ugly ones (multi-column layouts, tables, headers/footers,
  scanned-ish exports). Target: 10+ files.
- `jds/` — real job descriptions as plain `.txt`, pasted the way users paste
  them (boilerplate, bullet soup, and all). Target: 5+ files.

Rules:

- **These files are fixtures. Never edit them to make tests pass.** If a parser
  fails on a corpus file, the parser is wrong, not the file.
- Files prefixed `synthetic_` were generated by
  `scripts/make_synthetic_corpus.py` (fictional people, deliberately awkward
  layouts) and are placeholders until enough real files land. Regenerate them
  only via that script.
- Don't commit files containing real personal data of anyone who hasn't agreed
  to be in a public test corpus — scrub names/emails/phones first if in doubt.
